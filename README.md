## 1. Objetivo do Semestre

Minha principal motivação nesta disciplina é aprender formas simples de resolver problemas complexos, através de desafios de lógica da plataforma LeetCode e outras da mesma ordem. Profissionalmente, quero estar preparado para processos seletivos de Big Techs, onde a eficiência do algoritmo é critério de corte.


## 2. Diagnóstico Inicial


### 2.1. Análise de complexidade

#### O que já sei fazer

Eu sei quais são as complexidades que existem, tanto de espaço quanto de tempo (até porque são compartilhadas a maioria).

#### O que ainda gera dificuldade

Preciso melhorar minha análise de complexidade, muitas coisas eu já pensei que eram O(1) quando eram O(n) ou vice-versa.


### 2.2. Divisão e conquista

#### O que já sei fazer

Entendo o conceito de divisão e conquista, já implementei alguns algoritmos que usam isso.

#### O que ainda gera dificuldade

Ainda possuo problemas em aplicar o conceito em problemas novos que não sejam de ordenação.


### 2.3. Recursão

#### O que já sei fazer

Compreendo o caso base e a chamada recursiva em estruturas simples.

#### O que ainda gera dificuldade

Ainda está difícil perceber quando usar a recursividade, uso muito laços de repetição.


### 2.4. Programação Dinâmica

#### O que já sei fazer

Sei quebrar a maioria dos problemas maiores em subproblemas simples.

#### O que ainda gera dificuldade

Ainda possuo dificuldade em identificar a relação de recorrência e decidir entre bottom-up ou top-down.


### 2.5. Estruturas de dados

#### O que já sei fazer

Domino listas, pilhas e filas.

#### O que ainda gera dificuldade

A dificuldade está em decidir uma estrutura para o exercício. Também tenho problemas com algumas outras estruturas, como Heap


### 2.6. Grafos

#### O que já sei fazer

Sei apenas o conceito de grafos, já mexi com algumas formas de grafos como o LangGraph para criar agentes de IA, mas não me aprofundei muito.

#### O que ainda gera dificuldade

Ainda é necessário entender como os grafos funcionam para poder implementar os algoritmos que necessitam dele, como o de Dijkstra ou A*.


## 3. Metas técnicas

### 3.1. Especialização em Programação Dinâmica (DP)

- Contexto: É um dos tópicos mais desafiadores e recorrentes em competições e entrevistas.
- Descrição: Quero desenvolver a habilidade de Programação Dinâmica.
- Plano de Ação: Resolver pelo menos 10 exercícios de DP do LeetCode.
- Medição: Quantidade de exercícios sobre DP resolvidos no LeetCode.
- Prazo: até 1 de maio.


### 3.2. Implementação Robusta de Grafos

- Contexto: Grafos modelam a maioria dos problemas reais de redes e logística.
- Descrição: Quero melhorar minha habilidade com Grafos.
- Plano de Ação: Resolver pelo menos 10 problemas de grafos no LeetCode.
- Medição: Passar nos problemas do LeetCode com uma complexidade aceitável.
- Prazo: Final do semestre.

### 3.3. Melhorar na tomada de decisão de uso das estruturas de dados

- Contexto: É importante saber qual é a melhor estrutura de dados para cada problema.
- Descrição: Quero aprimorar minha habilidade de identificação.
- Plano de Ação: Resolver pelo menos 30 problemas sortidos do LeetCode.
- Medição: Passar nos problemas do LeetCode.
- Prazo: Final do semestre.

## 5. Plano de uso de IA

### 5.1. Como pretende usar IA para aprender

Utilizarei a IA como um tutor. Em vez de pedir o código pronto, pedirei explicações sobre o funcionamento de uma lógica ou para identificar onde está o erro no meu raciocínio.

### 5.2. Em quais momentos será apoio

Quando encontrar um erro e não conseguir visualizar onde ele está, nesse caso pedirei para me apontar o que está errado, mas sem mais dicas.

### 5.3. Quais limites vai impor para evitar uso automático ou superficial

- Proibido copiar e colar enunciados para obter a solução completa.
- Sempre que a IA sugerir uma otimização, devo ser capaz de explicá-la linha por linha sem o apoio da ferramenta.
- Uso restrito apenas após pelo menos 40 minutos de tentativa individual no problema.
